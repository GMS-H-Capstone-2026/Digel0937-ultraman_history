## 🛠️ [준비 단계] 로컬 서버 구동 및 진단

1. **Live Server 실행**: 
   * VS Code 우측 하단의 `Go Live` 버튼을 누르거나, `index.html` 파일 우클릭 후 **Open with Live Server**를 선택하여 웹 브라우저로 사이트를 실행합니다. (주소 예시: `http://127.0.0.1:5500/index.html`)
2. **Lighthouse 리포트 생성**:
   * 브라우저에서 `F12`를 눌러 개발자 도구를 엽니다.
   * 상단 탭에서 **Lighthouse**를 찾습니다 (보이지 않으면 `>>` 모양 화살표 클릭).
   * 기기를 **데스크톱(Desktop)**으로 선택하고, 카테고리는 모두 체크한 뒤 **페이지 로드 분석(Analyze page load)** 버튼을 누릅니다.
   * 생성된 리포트의 최초 점수를 위의 **점수 기록판**에 기록합니다.

## ⚡ [미션 1] Lighthouse 웹 최적화 (성능 90점 돌파하기)

Lighthouse 보고서 하단에 뜬 **개선 기회(Opportunities)**와 **진단(Diagnostics)** 목록을 분석하고, AI 에이전트와 소통하여 사이트를 최적화합니다.

### 1-1. 이미지 최적화 (성능 개선의 핵심)
* **문제점**: 초대형 JPG/PNG 이미지들이 로딩 속도를 갉아먹고 있습니다.
* **해결 팁**:
  * 이미지들을 차세대 포맷인 **WebP** 형식으로 변경하고 파일 크기를 압축합니다.
  * 무료 온라인 변환 사이트(예: [Squoosh](https://squoosh.app/))나 AI에게 변환 방법을 문의하여 이미지를 WebP로 가공합니다.
  * 이미지 태그(`<img>`)에 `loading="lazy"` 속성을 추가하여 화면에 보일 때만 이미지를 로드하게 합니다.
* **바이브 코딩 프롬프트 예시**:
  ```text
  "Lighthouse 성능 점수를 올리려고 해. index.html 파일 내의 이미지 태그들에 lazy loading 속성을 추가하는 코드를 수정해줘. 그리고 styles.css에서 카드 배경 이미지들이 로딩될 때 성능을 떨어뜨리지 않게 최적화하는 CSS 팁과 수정할 클래스 코드블록을 보여줘."
  ```

### 1-2. 접근성(Accessibility) 및 권장사항 보완
* **문제점**: 스크린 리더기나 장애인을 위한 접근성 설명(`aria-label`)이 부족하거나 시맨틱 태그 구조가 꼬여 있습니다.
* **해결 팁**:
  * 소리가 켜지고 꺼지는 사운드 버튼, 플레이어 제어 버튼 등에 명확한 설명 속성을 추가합니다.
  * `<header>`, `<nav>`, `<main>`, `<footer>` 등의 HTML5 시맨틱 태그 배치가 올바른지 점검합니다.
* **바이브 코딩 프롬프트 예시**:
  ```text
  "사운드 온오프 버튼이랑 음악 플레이어 제어 버튼들에 시각 장애인을 위한 aria-label 속성이 빠져 있어서 Lighthouse 점수가 낮게 나와. index.html의 해당 버튼 코드 부분을 찾아서 적절한 aria-label을 추가해줘. 전체 파일 말고 수정한 HTML 버튼 코드만 추출해서 보여줘."
  ```

### 1-3. 검색 엔진 최적화 (SEO) 극대화
* **문제점**: 메타 정보와 타이틀 설명이 빈약하여 검색 엔진에 잘 걸리지 않는 구조입니다.
* **해결 팁**:
  * `<head>` 안에 사이트 정보(description, keywords, author) 메타 태그를 명확히 추가합니다.
* **바이브 코딩 프롬프트 예시**:
  ```text
  "빛의 전설: 울트라맨 역사 박물관 사이트에 맞는 SEO 검색 최적화 meta 태그들을 만들어서 index.html <head> 부분에 삽입해줘. 수정된 head 영역 코드블록만 diff 형식으로 제공해줘."
  ```

## 💾 [미션 2] 데이터 JSON 분리 및 Fetch API 비동기 로딩

현재 `app.js`는 울트라맨과 괴수들의 방대한 텍스트 데이터가 하드코딩되어 파일 크기가 74KB에 이릅니다. 이 방대한 텍스트 데이터를 JSON 파일로 격리하고, 비동기 API 통신을 통해 화면에 동적으로 렌더링되도록 리팩토링합니다.

### 2-1. JSON 파일로 데이터 격리하기
* 프로젝트의 `data/` 폴더 내에 생성되어 있는 세 개의 JSON 파일에 데이터를 옮깁니다:
  1. `data/ultramen.json` : `app.js` 맨 위의 `ULTRAMAN_DATA` 배열 전체를 복사해서 붙여넣습니다.
  2. `data/kaijus.json` : `app.js` 내의 `KAIJU_DATA` 배열을 복사해서 붙여넣습니다.
  3. `data/songs.json` : `app.js` 내의 `SONG_PLAYLIST` 배열을 복사해서 붙여넣습니다.
* **⚠️ 주의 (JSON 포맷 규칙)**:
  * JS 객체 리터럴과 달리 JSON 파일은 모든 Key값과 문자열 Value값을 **반드시 큰따옴표(`"`)**로 감싸야 합니다. 싱글 쿼트(`'`)는 사용할 수 없습니다.
  * 마지막 요소 뒤에 쉼표(`,`)가 붙으면 문법 에러가 발생합니다.
  * *팁*: AI에게 `"app.js에 있는 ULTRAMAN_DATA 배열을 유효한 JSON 포맷 데이터로 완전히 변환해서 텍스트로 보여줘"`라고 요청하여 간편하게 변환본을 받으세요.

### 2-2. Fetch API 연동 및 비동기 렌더링 구현
* `app.js` 상단에서 하드코딩되었던 `ULTRAMAN_DATA`, `KAIJU_DATA`, `SONG_PLAYLIST` 변수 선언 부분을 과감히 지웁니다.
* 대신 브라우저가 기동될 때 `data/` 폴더의 JSON 파일들을 비동기(`fetch`, `async/await`)로 읽어와 기존 변수에 할당하고 화면 렌더링을 시작하도록 자바스크립트를 수정합니다.
* **바이브 코딩 프롬프트 예시**:
  ```text
  "app.js에서 기존에 선언되어 있던 ULTRAMAN_DATA, KAIJU_DATA, SONG_PLAYLIST 배열을 전부 지웠어. 대신 페이지가 로드될 때 fetch API를 이용해 'data/ultramen.json', 'data/kaijus.json', 'data/songs.json' 파일로부터 비동기(async/await)로 데이터를 로드하여 전역 변수에 담고, 기존의 renderUltramen(), renderKaijus(), initMusicPlayer() 함수들을 실행하게 app.js의 초기 로드 로직을 수정해줘."
  ```

### 2-3. 데이터 로딩 대기 연출 (선택 미션)
* 로컬 환경은 데이터가 금방 받아와지지만, 실제 서버 배포 시 네크워크 지연으로 데이터 로딩이 수 초간 걸릴 수 있습니다.
* 데이터 로딩 중에는 화면에 **"M78 우주경비대 데이터베이스 로딩 중..."** 메시지와 함께 부드러운 스피너(Spinner) 애니메이션이 돌다가, 로딩이 완료되면 실제 카드가 켜지도록 연출해 봅니다. (AI에게 CSS 로딩 인디케이터 제작 및 제어 로직을 요청해 보세요!)

## 💡 Gemini 무료 계정을 위한 토큰 세이브 바이브 코딩 팁

토큰 한도가 작은 Gemini 무료 체험 계정으로 긴 수업 시간 동안 오류 없이 코딩하기 위해 반드시 숙지해야 하는 규칙입니다.

1. **diff(차이점) 또는 변경 부분만 응답하게 하기**
   * "코드를 수정해줘"라고만 하면 AI가 수백 줄의 파일 전체를 다시 출력하여 금방 토큰이 마릅니다.
   * 반드시 **"전체 코드를 출력하지 말고, 바뀐 특정 함수 부분만 리턴해줘"** 혹은 **"수정해야 할 스타일 클래스 영역만 CSS 코드로 제공해줘"**라고 단서를 다세요.
2. **콘솔 에러를 활용한 디버깅**
   * 코드가 동작하지 않을 때는 무작위로 "안돼, 고쳐줘" 하지 마세요.
   * `F12` 개발자 도구의 **Console(콘솔)** 탭에 뜬 빨간색 에러 로그(텍스트 전체)를 그대로 복사하여 AI에게 제공해야 단 1번의 질문으로 완벽한 수정 코드를 얻을 수 있습니다.
   * *예시*: `"app.js 1192라인에서 'TypeError: Cannot read properties of null (reading 'textContent')' 에러가 나. 해당 함수 코드와 이 에러 메시지를 줄 테니 원인을 분석해서 교체할 함수 코드만 줘."`
